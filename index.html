SHC Application Framework

Das Application Framework ist der Nachfolger des Raspberry Pi SmartHome Control 2.x. Das Apllication Framework wird zu großen Teilen in Java geschrieben und bricht damit einige grenzen des in PHP geschriebenen RPi SmartHome Controls auf.

Module

Das Framework besteht aus mehreren Modulen. Im folgenden sind die Module kurz aufgeführt und erläutert.

SHC Core

Der Core ist die Basis Libary weleche für alle anderen Module die gemeinsamen Daten und Funktionen zusammenführt.

Application Server

Der Appliaction Server ist da neue Herz des Framework. Er Verwaltet alle Daten und bearbeitet die Automatisierungsfunktionen. Zudem stellt der Server alle nötigen Schnittstellen für die Grafikoberflächen bereit. Gespeichert werden Alle Daten in eine Redis Datenbank. Alle Benutzeroberflächen verbinden sich mit der Server und dienen nur noch zur Anzeige und Interaktion mit dem Benutzer.

CLI Client

Der CLI Client ist ein Kommandozeilen Tool über das sich der Status der Elemente abfragen lässt und die schaltbaren Elemente geschaltet werden können.

Desktop Client

Der Desktop Client ist die neuen Hauptoberfläche des SHC. Über ihn lassen sich allen Funktionen des SHC verwenden und Einstellen. Über ihn lassen sich auch alle Verwaltungsfunktionen verwenden. Der Desktop Client wird in JavaFX 8 geschrieben und setzt Java 8 Update 40 voraus.

Android Client

Der Android Client wird einen ähnlichen Funktiontionsumfang wie der Desktop Client erhalten, nur speziell auf die Bedingung über Mobile Endgeräte ausgelegt. Vorgesehen ist hier eine Unterstützung ab Android 4.0.

Web Client

Der Web Client ist eine Weboberfläche die für Desktop und Mobile Endgeräte optimiert ist. Allerdings können über den Web Client keine Verwaltungsfunktionen mehr verwendet werden.

Schaltserver

Der Schaltserver dient wie im alten SHC als Schnittstelle zur Hardware. Zudem wird er im Funktionsumfang noch einmal deutlich erweitert. Wie schon zuvor können mehrere Schaltserver verwendet werden.

Sensorsender

Der Sensorsender wird im im alten SHC die Sensordaten auf dem Raspberry Pi an den Server schicken.

Unterstützte Funktionen

schaltbare/lesbare Elemente

  • 433MHz Steckdosen
  • AVM DECT Steckdosen
  • Edimax SP-1101W und SP2101W WLan Steckdosen neu
  • Fritz!Box Funktionen (WLan an/aus, Neustart ...)
  • Neustart/Herunterfahren des Servers
  • Neustart/Herunterfahren von Schaltservern neu
  • GPIOs vom Raspberry Pi und Microcontrollern als Eingang lesen und als Ausgang schalten
  • Scripte auf dem App Server oder Schaltserbern ausführen neu
  • Rechner im Netzwerk starten "Wake on Lan"
  • Benutzer zu Hause

Gruppen Elemente

  • Aktivitäten (Gruppenschaltung von mehreren schaltbaren Elementen)
  • Countdown (zeitlich begenzte Gruppenschaltung von mehreren schaltbaren Elementen)
  • Button (Tastfunktion)

Sensoren

  • DS12x20 Sensoren
  • DHT Sensoren
  • BMP Sensoren
  • Regen Sensoren
  • Feuchtigkeits Sensoren
  • Lichtstärke Sensoren
  • Wasserzähler, Gaszähler Stromzähler neu
  • HC-SR04 Sensoren neu
  • SCT-013 Sensoren neu
  • Akkus (Ladezustand) neu

virtuelle Sensoren (Statistische Mittelwerte von Sensoren mit gleichen Werten)

  • Energiemesser neu
  • Stromverbrauch neu
  • Feuchtigkeit neu
  • Lichtstärke neu
  • Temperatur neu
  • Luftfeuchte neu
  • Menge (Strom, Gas, Wasser) neu
